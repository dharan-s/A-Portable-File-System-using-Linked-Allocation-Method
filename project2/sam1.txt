1.Period Source Generation :
  - Copy rule tables from 'MSM' schema into 'PRD_REFMFRP' schema
  -     1.PRD_M_<Cycle Date>
        2.PRD_M_P_<Cycle Date>
        3.PRD_SBST_<Cycle Date>

2.Incoming
  - copies tables from Period Schema in Oracle to Netezza stage database
  - 'PRD_REFMFRP'
    PRD_M_<cycle-date>,
    PRD_M_P_<cycle-date>,
    PRD_SBST_<cycle-date>,
    DIM_RECEIPT(audit table in which is populated by the incoming module For each dimension )
	
3.Adapter :
  - The custom adapter process for period dimension will populate the STG_DIM_CHR_VAL_PRD   table
    in period schema that can easily be referenced by the metadata management process	
  - 'PRD_REFMFRP'
    STG_DIM_CHR_VAL_PRD 

4.MSM Refresh Load
  - read the data from the common data store, calculate the new and update dimension characteristics and
    send the list of unique characteristics and values to MSM  
  - retrieve a CHR_ID which will be generated by MSM to be later applied in the dimension table
  - also send characteristic attributes ( such as data type, precision, display flag etc.. ), 
    so MSM can create the associated build instructions and database delivery information.  
  - 'MSM Server'
    STG_DIM_CHR_PRD 
	STG_DIM_CHR_DT_SRC_PRD 
	STG_DIM_CHR_LOV_PRD
